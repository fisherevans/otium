otium — visual simplification

A pass to cut visual noise: the app leans on hard borders and boxed chips almost everywhere, which reads as "a panel of controls" and fights calm reading. This audits the frivolous elements and mocks up de-noised versions of the key pages. July 2026.

The core move: replace borders and boxes with typography and whitespace. A bordered box says "I'm a control, look at me." A quiet label with the right size/weight/space says the same thing without shouting. Reserve borders for genuine inputs (a text field), and reserve the one solid element per screen for the primary action.

3 · The rules (once, applied everywhere)

  1. Type & space over borders. Hierarchy comes from size / weight / caps / color / whitespace — not from drawing a box around things.
  2. Active state = fill or underline, not a persistent border. Don't outline every option; mark only the chosen one.
  3. One solid element per screen — the primary action. Everything else is quiet. If two things are heavy, one is wrong.
  4. Icons live free. No bordered button-boxes, especially in the reader. A padded tap target, not a drawn frame.
  5. Faint dividers, or none. Whitespace separates rows; reserve a visible rule for a real section break.
  6. Keep management detail off the reading surface. Weight badges, score cues, and steppers belong in management screens, not on the card you're reading.
  7. Borders are earned — a text input or a deliberately-destructive action (Delete) may keep one. A label never does.
Mockups are schematic — they show the treatment (border vs. no border, boxed vs. typographic), not final pixels. This is a token-level change (border-widths, chip/badge/segment styles, divider weights, action-bar chrome) plus pulling management detail off the card — so it's mostly CSS, low-risk, and lands screen by screen with a Palma-res check each time.
